ZERO "todo" comments or stub sections. Every feature described below must be fully implemented, wired end-to-end, and functional. If a section says "cart drawer," build the complete cart drawer — not a placeholder div with a comment saying "add cart logic here."
Production-grade or nothing. This is not a prototype. This is not a template. This is a shipped product. Every interaction must feel finished, every edge case handled, every state accounted for.

3. TECHNICAL ARCHITECTURE
3.1 Stack
Framework: Next.js 14+ (App Router)
Styling: Tailwind CSS v3.4+ with the design tokens mapped to tailwind.config.js theme extensions. CSS Modules for complex component-specific styles that Tailwind can't express elegantly. NO inline styles, ever.
Animations: GSAP 3.12+ (ScrollTrigger, SplitText-equivalent via SplitType), Framer Motion 11+ (layout animations, AnimatePresence for page/mount transitions, gesture-based interactions)

Carousels: Embla Carousel (product image galleries, category carousels, testimonial carousels, related product carousels)

Icons: Lucide React
Image Optimization: Next.js <Image> with proper sizes, priority, placeholder="blur", and blurDataURL
State Management: React Context + useReducer for cart/wishlist. Zustand is acceptable as an alternative.
Form Handling: React Hook Form + Zod validation
Data: Static JSON/TypeScript data files with full product catalog (minimum 24 products across all categories), collections, testimonials, FAQ data

4. ANIMATION SYSTEM — DETAILED SPECIFICATIONS
This is the heart of the premium feel. Every animation must be purposeful, performant, and polished.

4.1 Lenis Smooth Scrolling Setup
Initialize Lenis in a client-side provider wrapping the entire app
Duration: 1.2s, easing: 0.075, 0.1, 0.1, 1 (smooth ease-out)
Sync with GSAP ScrollTrigger: lenis.on('scroll', ScrollTrigger.update) and gsap.ticker.add((time) => lenis.raf(time * 1000))
Disable on mobile when user prefers-reduced-motion is set
Smooth scroll to anchors when navigation links are clicked
Scroll to top on route change
4.2 Text Animations (SplitType + GSAP)
SplitTextReveal Component:

Accepts props: text, type ('char' | 'word' | 'line'), animation ('slideUp' | 'fadeIn' | 'blurIn' | 'stagger'), stagger (number), duration (number), delay (number), trigger ('scroll' | 'mount')
On scroll trigger: uses ScrollTrigger with start: 'top 85%', once: true
On mount trigger: plays immediately with optional delay
Default stagger: 0.03s for chars, 0.06s for words, 0.1s for lines
Default duration: 0.8s
Easing: power3.out
TextBlurReveal Component:

Text starts with opacity: 0, filter: blur(10px), transform: translateY(15px)
Animates to opacity: 1, filter: blur(0px), transform: translateY(0)
Duration: 0.9s, easing: power2.out
Can be used on individual elements or as a stagger parent
Scroll-triggered by default
Headline Animation Standard:

Every H1 on the site uses SplitTextReveal with type='line', animation='slideUp', stagger=0.12
Every H2 on the site uses TextBlurReveal
Every H3 uses RevealOnScroll with direction='up', distance=20
4.3 Scroll-Triggered Reveals (RevealOnScroll Component)
A versatile wrapper component:

interface RevealOnScrollProps {  children: React.ReactNode  direction?: 'up' | 'down' | 'left' | 'right' | 'none'  distance?: number       // default: 30  duration?: number       // default: 0.8  delay?: number          // default: 0  stagger?: number        // staggers direct children, default: 0.1  blur?: boolean          // adds blur to the reveal, default: false  once?: boolean          // default: true  start?: string          // ScrollTrigger start, default: 'top 85%'  scroller?: string       // for nested scroll containers}

4.4 Page Transitions (Framer Motion AnimatePresence)
Wrap page content in <PageTransition> component
Exit animation: content fades out and slides up 20px, duration 0.3s

Header and footer are OUTSIDE the page transition wrapper — they persist

Layout animation: When filtering causes products to rearrange, use Framer Motion layout prop for smooth position transitions
4.6 Cart Drawer Animation (Framer Motion)
Drawer slides in from right: x: '100%' → x: 0, duration 0.5s, easing: [0.16, 1, 0.3, 1]

Key responsive behaviors:

Header: 60px mobile → 72px desktop

9. PERFORMANCE & OPTIMIZATION

Embla: Destroy carousel instances on unmount
